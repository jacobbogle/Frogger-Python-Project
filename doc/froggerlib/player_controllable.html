PlayerControllable¶

Defines how players will interact with objects in the game. All classes that define objects that are to be controlled by the player should inherit (whether directly or indirectly) from the PlayerControllable class.

You shouldn’t directly create an instance of PlayerControllable but consider the descriptions below of the data attributes/members and methods that are accessible on any object that inherits from PlayerControllable.

class PlayerControllable¶
horizontal_gap¶

A value representing how far the controlled object will horizontally move on a single command (i.e., how far the frog will jump when pressing an arrow key).

vertical_gap¶

A value representing how far the controlled object will vertically move on a single command (i.e., how far the frog will jump when pressing an arrow key).

getHorizontalGap()¶
Returns

The horizontal gap value.

Return type

int or float

getVerticalGap()¶
Returns

The vertical gap value.

Return type

int or float

setHorizontalGap(hg)¶
Parameters

x (int or float) – The value to set for the horizontal gap of the object.

setVerticalGap(y)¶
Parameters

x – The value to set for the vertical gap of the object.

up()¶

Command the controlled object to move up in the window. Needs to be paired with a call to Movable.move() where up() is called when the player presses the up key, and Movable.move() is called when the game evolves.

down()¶

Command the controlled object to move down in the window. Needs to be paired with a call to Movable.move() where down() is called when the player presses the down key, and Movable.move() is called when the game evolves.

left()¶

Command the controlled object to move left in the window. Needs to be paired with a call to Movable.move() where left() is called when the player presses the left key, and Movable.move() is called when the game evolves.

right()¶

Command the controlled object to move right in the window. Needs to be paired with a call to Movable.move() where right() is called when the player presses the right key, and Movable.move() is called when the game evolves.

outOfBounds(screen_width, screen_height)¶

Reports whether or not the object has moved off of the screen. This is determined based on the screen size parameters.

Parameters

  • screen_width (int or float) – The width of the screen/window.

  • screen_height (int or float) – The height of the screen/window.

Returns

Whether or not the object has moved off of the screen.

Return type

bool
